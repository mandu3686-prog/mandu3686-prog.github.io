## 🏗️ 이 사이트를 만든 방법: 첫 번째 프로젝트

이 포트폴리오 사이트는 정적 사이트 생성기(Static Site Generator)를 활용한 웹사이트 구축 과정을 학습하기 위한 첫 번째 프로젝트입니다.

### 📌 사용 기술 및 환경
* **배포 플랫폼**: GitHub Pages (정적 웹사이트 호스팅)
* **프레임워크**: Jekyll (Markdown 기반의 정적 사이트 생성기)
* **테마**: So Simple Theme (YAML/Markdown 문법 학습을 위한 미니멀 테마 선택)
* **주요 사용 파일**: `_config.yml` (사이트 설정), `index.md` (메인 콘텐츠), `assets/css/style.css` (색상 커스터마이징)

### 💡 제작 과정에서 배운 핵심
| 학습 주제 | 구체적 경험 및 배운 점 |
| :--- | :--- |
| **YAML 문법** | `_config.yml` 설정 시 콜론 뒤 공백 누락 등으로 빌드 실패(404 에러)를 반복 경험했습니다. 이 과정을 통해 **문법의 정확성**과 **디버깅 능력**의 중요성을 체감했습니다. |
| **테마 활용** | `remote_theme` 설정만으로 사이트 디자인을 변경하고, CSS 파일을 추가하여 **기존 테마 위에 새로운 스타일을 덮어쓰는** 방법을 학습했습니다. |
| **버전 관리** | 사이트 제작 초기부터 Git의 커밋(Commit)과 푸시(Push)를 통해 **변경 이력 관리**를 습관화했습니다. |

## ⏭️ 앞으로의 방향성 (Next Steps)

현재의 성장에 만족하지 않고, 앞으로 집중적으로 학습하고 개선할 영역을 명확히 제시합니다.

### 1. 기술적 심화 목표
* **[프레임워크 심화]**: 현재 **React** 기초 학습에서 나아가, **상태 관리 라이브러리 (Redux/Zustand)**를 심화 학습하여 [복잡한 기능]을 구현하는 프로젝트에 적용해 볼 계획입니다.
* **[기본기 보강]**: **바닐라 JavaScript**를 사용하여 웹 API(예: Geolocation API, Web Storage)를 다루는 연습을 통해 기초 지식을 탄탄하게 다질 것입니다.
* **[성능 최적화]**: 현재 사이트의 로딩 속도나 이미지 최적화 등의 문제를 파악하고 개선하는 과정을 다음 목표로 설정하겠습니다.

### 2. 포트폴리오 개선 목표
* **[페이지 확장]**: 현재 `index.md` 위주에서 벗어나, `projects.md`와 `about.md`에 **맞춤형 레이아웃**을 적용하여 정보 접근성을 높이겠습니다.
* **[UI/UX 개선]**: 사용자 정의 CSS를 더 깊이 학습하여, [현재 설정한 강조 색상] 외에도 더 전문적이고 일관된 디자인 시스템을 적용해 볼 계획입니다.
